# toss-stock-simulator

토스증권 Open API 시세 데이터를 사용해 **라오어의 무한매수법**을 시뮬레이션하는 백테스트/시뮬레이터.

## 목표

과거(또는 실시간) 시세를 입력으로, 무한매수법 규칙을 그대로 기계적으로 실행했을 때의
매수 체결 내역 · 평단 변화 · 익절 시점 · 최종 수익률을 재현한다.

- 규칙은 하드코딩하지 않고 파라미터화 (분할 수, 익절률, 종목 등)
- 실주문은 실행하지 않는다. 시뮬레이션 전용.

## 무한매수법 규칙 요약

3배 레버리지 ETF(TQQQ, SOXL 등)를 대상으로 하는 기계적 분할매수 → 익절 → 반복 전략.
버전이 여러 개 있고 규칙이 다르다. **시뮬레이터는 버전을 선택 가능하게 구현한다.**

### 용어

| 용어 | 뜻 |
|---|---|
| **T (회차)** | 진행된 매수 회차. `누적 매수액 / 1회 매수액`. 버전에 따라 부분 매도로 줄기도 한다 |
| **별% / 별지점** | 매수와 매도를 가르는 기준선(평단 대비 %). 별지점 아래는 매수 영역, 위는 매도 영역 |
| **LOC** (Limit On Close) | 종가 동시호가 주문. 매수는 종가 ≤ 지정가, 매도는 종가 ≥ 지정가일 때 **종가로** 체결 |
| **LOC평단 매수** | 평단 가격에 거는 LOC 매수 |
| **LOC큰수 매수** | 평단보다 높은 값(별지점 근처)에 거는 LOC 매수. 상승일에도 매수가 걸리게 하는 장치 |
| **쿼터매도** | 보유 수량의 1/4을 정리하는 규칙. v3.0부터 '쿼터손절'에서 이 이름으로 통일 |
| **잔금** | 아직 매수에 쓰지 않은 현금 |
| **전반전 / 후반전** | 분할 진행 정도 구분. v4.0 기준 후반전은 별%가 음수인 구간 (`T ≥ 분할수/2`) |
| **리버스모드** | v4.0에서 정리된, 분할 자금을 다 소진한 국면. 무한매도 + 쿼터매수로 대응 |

### 공통 개념

- 3배 레버리지 ETF를 매일 분할 매수해 평단을 끌어내리고, 반등 시 익절 후 사이클을 다시 시작한다
- 매수·매도 모두 **미리 예약 주문을 걸어두고 그날 종가/고가로 체결 여부가 갈린다**
- 사이클 종료 판정은 버전마다 다르다. v4.0은 **보유수량 0** 기준 (익절 여부가 아님)

### v1 (기본형)

1. 총 시드를 40등분. 매일 1/40씩 매수
2. 매수: 평단가를 상한으로 LOC 매수 (평단보다 비싸면 안 삼)
3. 매도: 평단가 × 1.1 지정가 매도 예약
4. 40회 소진 후에도 미익절이면 보유 수량 1/4을 시장가 매도해 현금 확보, 사이클 재개

### v2.1

LABU 사태를 계기로 나온 버전. 전반전/후반전을 나누고,
일부 물량을 +5% 수준에서 익절해 현금흐름을 만들어 안정성을 높인다.
(세부 수식은 원저작 서적 기준 — 구현 전 출처 확인 필요)

### v2.2 (40분할, 단리)

- **전반전 (T < 20)**
  - 1회 매수액의 1/2 → 평단가 LOC 매수
  - 나머지 1/2 → 평단 대비 `+(10 - T/2)%` LOC 매수
- **후반전 (T ≥ 20)**
  - 1회 매수액 전체 → 평단 대비 `(10 - T/2)%` LOC 매수
- **매도 (전·후반전 동일)**
  - 누적 수량의 1/4 → 평단 대비 `(10 - T/2)%` LOC 매도
  - 나머지 3/4 → 평단 +10% 지정가 매도

### v3.0 (20분할, 복리)

- **전반전 (T ≤ 19)**
  - 1회 매수액의 1/2 → 평단 대비 `(15 - 1.5T)%` LOC 매수
  - 나머지 1/2 → 평단가 LOC 매수
- **후반전 (T > 19)**
  - 1회 매수액 전체 → 평단 대비 `(15 - 1.5T)%` LOC 매수
- **매도**
  - 누적 수량의 1/4 → `(15 - 1.5T)%` LOC 매도
  - 나머지 3/4 → TQQQ +15%, SOXL +20% 지정가 매도
- **쿼터모드 (19 < T < 20)** — 1/4 MOC 매도, 3/4 지정가 매도. 이날은 매수 없음
- 수익금을 다음 사이클 시드에 즉시 반영 (복리)

### v4.0 (20/30/40분할) — **현재 구현 대상**

공개 자료만으로 추측 없이 구현 가능한 유일한 버전이라 이걸 먼저 만든다.
일반모드와 리버스모드 두 국면으로 나뉜다.

#### 일반모드

```
별%       = base × (1 − 2T/분할수)      base: TQQQ 15, SOXL 20
별지점    = 평단 × (1 + 별%/100)
매수가    = 별지점 − 1틱 ($0.01)
매도가    = 별지점
1회매수금 = 잔금 / (분할수 − T)
```

- 출처는 종목·분할별로 따로 적어두지만(`15−0.75T`, `15−1.5T`, `20−T`, `20−2T`)
  위 일반식으로 전부 재현되고, `T = 분할수/2`에서 별% = 0 이라
  "후반전 = 별% 음수" 정의와도 맞아떨어진다. 덕분에 30분할도 커버된다.

**T 전이**

| 사건 | T |
|---|---|
| 1회분 매수 체결 | +1 (절반이면 +0.5) |
| 쿼터매도(1/4) 체결 | × 0.75 |
| 지정가매도(3/4) 체결 | × 0.25 |

부분 매도는 **남은 수량 비율만큼 T를 줄인다**로 일반화된다. T가 줄면
`잔금/(분할수−T)` 공식상 매수 여력이 회복된다.

**하루 주문**

- 첫날 (T=0, 보유 0) — 전일 종가 위에서 큰수 LOC 매수
- 전반전 (T < 분할수/2) — 1회매수금의 절반은 별지점 아래, 절반은 평단에 LOC 매수
- 후반전 (T ≥ 분할수/2) — 1회매수금 전액을 별지점 아래에 LOC 매수 (평단보다 낮음)
- 매도 (보유 > 0) — 1/4은 별지점 LOC 매도, 3/4은 평단 +base% 지정가 매도

**사이클 종료 = 보유수량 0.** 익절했다고 끝나는 게 아니다.
3/4만 익절되고 1/4이 남으면 사이클은 이어진다. 지정가 매도 후 급락으로
같은 날 LOC 매수가 체결되는 것도 정상 동작이다.

#### 리버스모드

원금을 다 소진한 국면. 일반모드와 방향이 뒤집혀 **무한매도 + 쿼터매수**로 버틴다.

- **진입** — `T > 분할수 − 1` (20분할이면 T > 19, 40분할이면 T > 39).
  잔금이 남아 있으면 리버스모드에서 함께 쓴다.
- **별지점** — 일반모드와 정의가 다르다. **직전 5거래일 종가 평균**.

**매도 (무한매도)**

| 시점 | 규칙 |
|---|---|
| 첫날 | 무조건 MOC 매도. 수량 = 전체 × `2/분할수` (20분할 1/10, 40분할 1/20), 내림 |
| 둘째날 이후 | 별지점 **위**에서 매도. 직전 보유 개수에 같은 비율을 적용 |

**매수 (쿼터매수)**

- 첫날은 매수 없음
- 둘째날부터 별지점 **아래**에서 `총 잔금 / 4` 규모로 LOC 매수
- 잔금에는 미체결 매도금도 포함한다

**T 전이**

| 사건 | T |
|---|---|
| 매도 후 | `T × (1 − 2/분할수)` (20분할 ×0.9, 40분할 ×0.95) |
| 매수 후 | `T + (분할수 − T) × 0.25` |

매도 계수 `1 − 2/분할수`는 곧 **남은 수량 비율**이다 —
일반모드의 T 축소 규칙과 같은 원리다.

**종료 (일반모드 복귀)**

종가가 평단 대비 TQQQ −15% / SOXL −20%보다 높으면 다음날 일반모드로 재시작한다.
T는 승계하고 `1회매수금 = 잔금 / (분할수 − T)`로 재계산한다.

### 버전 비교

| 항목 | v2.2 | v3.0 | v4.0 |
|---|---|---|---|
| 분할수 | 40분할 | 20분할 | 20/30/40 선택 |
| 1회매수금 | 시드/40 고정 | 시드/20 고정 | `잔금/(분할수−T)` — 매일 변동 |
| 별% 기준 | `10 − T/2` | `15 − 1.5T` | `base × (1 − 2T/분할수)` |
| TQQQ 매도 목표 | +10% | +15% | +15% (SOXL +20%) |
| 부분 매도 후 T | 자료 없음 | 자료 없음 | 남은 수량 비율로 축소 |
| 원금 소진 대응 | 매수액 감소 | 쿼터모드 | 리버스모드 (무한매도+쿼터매수) |
| 복리 | 단리 | 복리 | 선택 |
| 성격 | 안정적 생존 | 수익 극대화 | 국면별 대응 |

> 주의: 3배 레버리지 특성상 장기 하락장에서 손실도 3배로 확대된다.
> 본 프로젝트는 전략의 수익을 보장하지 않으며, 검증·학습 목적의 시뮬레이터다.

## 토스증권 Open API 요약

- REST API. 시세 / 종목정보 / 환율 / 시장정보 / 계좌 / 보유주식 / 주문 제공
- 인증: OAuth 2.0 Client Credentials Grant → access token (Bearer)
- 계좌·자산·주문 API는 `X-Tossinvest-Account` 헤더 추가 필요
- 지원 시장: 국내(KRX), 미국 주식
- 현재 사용 범위: **캔들(일봉) 시세, 종목정보, 환율** (읽기 전용).
  주문 API는 시뮬레이터 단계에서는 사용하지 않는다.

## 범위

- **현재**: 시뮬레이터만. 실제 주문 없음.
  - **백테스트** — 과거 일봉을 한 번에 흘려 결과 확인
  - **페이퍼 트레이딩** — 앱을 상시 띄워두고 매일 장 마감마다 하루치를 처리
- **추후**: 실주문 연동 가능성 있음. 그때까지 주문 관련 코드는 작성하지 않는다.

## 기술 스택

Node 24 + TypeScript. 런타임 의존성 없음.

| 레이어 | 선택 |
|---|---|
| 런타임 | Node ≥24 (내장 type stripping — 빌드 스텝 없이 `.ts` 직접 실행) |
| 타입체크 | `tsc --noEmit` (`npm run typecheck`) |
| 테스트 | `node:test` (`npm test`) |
| 저장소 | `node:sqlite` — 시세·시뮬 상태 모두 단일 DB 파일 |
| API 클라이언트 | 내장 `fetch` + OAuth2 토큰 캐시 |
| UI | (미착수) Vite + React SPA |

## 설정

```bash
npm install
cp .env.example .env   # TOSS_CLIENT_ID / TOSS_CLIENT_SECRET 입력

npm test
```

## 일봉 수집

```bash
node src/scripts/collect.ts TQQQ SOXL [--since 2020-01-01] [--full]
```

`GET /api/v1/candles` (`interval=1d`, 1회 200봉, `before` 로 과거 페이지네이션) 를 훑어
`DB_PATH` 의 `candles` 테이블에 저장한다. 기본은 증분 — 저장된 마지막 거래일부터 이어 받고,
같은 `(symbol, date)` 는 덮어쓰므로 같은 구간을 다시 받아도 결과가 같다.

## 백테스트 실행

```bash
node src/backtest/cli.ts <SYMBOL|candles.json> [--splits 40] [--seed 10000] [--simple] \
                        [--ticker TQQQ] [--from 2020-01-01] [--to 2024-12-31]
```

인자가 `.json` 이면 `{ date, open, high, low, close }` 배열 파일, 아니면 DB 에 수집된 심볼.
사이클별 손익, 리버스모드 일수, 최종 수익률, 최대 낙폭을 출력한다.
