# toss-stock-simulator

토스증권 Open API 시세 데이터를 사용해 **라오어의 무한매수법**을 시뮬레이션하는 백테스트/시뮬레이터.

## 목표

과거(또는 실시간) 시세를 입력으로, 무한매수법 규칙을 그대로 기계적으로 실행했을 때의
매수 체결 내역 · 평단 변화 · 익절 시점 · 최종 수익률을 재현한다.

- 규칙은 하드코딩하지 않고 파라미터화 (분할 수, 익절률, 종목 등)
- 실주문은 실행하지 않는다. 시뮬레이션 전용.

## 무한매수법 규칙 요약

3배 레버리지 ETF(TQQQ, SOXL 등)를 대상으로 하는 기계적 분할매수 → 익절 → 반복 전략.
버전이 여러 개 있고 규칙이 다르다. **시뮬레이터는 버전을 선택 가능하게 구현한다.**

공통 개념:

- **T** = 현재 사이클의 진행 회차 (= 소진한 분할 횟수, 1부터 시작)
- **LOC(Limit On Close)** — 지정가 이하일 때만 그날 종가로 체결, 아니면 취소
- 익절되면 전량 매도 → 사이클 종료 → 현금으로 새 사이클 시작

### v1 (기본형)

1. 총 시드를 40등분. 매일 1/40씩 매수
2. 매수: 평단가를 상한으로 LOC 매수 (평단보다 비싸면 안 삼)
3. 매도: 평단가 × 1.1 지정가 매도 예약
4. 40회 소진 후에도 미익절이면 보유 수량 1/4을 시장가 매도해 현금 확보, 사이클 재개

### v2.1

LABU 사태를 계기로 나온 버전. 전반전/후반전을 나누고,
일부 물량을 +5% 수준에서 익절해 현금흐름을 만들어 안정성을 높인다.
(세부 수식은 원저작 서적 기준 — 구현 전 출처 확인 필요)

### v2.2 (40분할, 단리)

- **전반전 (T < 20)**
  - 1회 매수액의 1/2 → 평단가 LOC 매수
  - 나머지 1/2 → 평단 대비 `+(10 - T/2)%` LOC 매수
- **후반전 (T ≥ 20)**
  - 1회 매수액 전체 → 평단 대비 `(10 - T/2)%` LOC 매수
- **매도 (전·후반전 동일)**
  - 누적 수량의 1/4 → 평단 대비 `(10 - T/2)%` LOC 매도
  - 나머지 3/4 → 평단 +10% 지정가 매도

### v3.0 (20분할, 복리)

- **전반전 (T ≤ 19)**
  - 1회 매수액의 1/2 → 평단 대비 `(15 - 1.5T)%` LOC 매수
  - 나머지 1/2 → 평단가 LOC 매수
- **후반전 (T > 19)**
  - 1회 매수액 전체 → 평단 대비 `(15 - 1.5T)%` LOC 매수
- **매도**
  - 누적 수량의 1/4 → `(15 - 1.5T)%` LOC 매도
  - 나머지 3/4 → TQQQ +15%, SOXL +20% 지정가 매도
- **쿼터모드 (19 < T < 20)** — 1/4 MOC 매도, 3/4 지정가 매도. 이날은 매수 없음
- 수익금을 다음 사이클 시드에 즉시 반영 (복리)

### 버전 비교

| 항목 | v2.2 | v3.0 |
|---|---|---|
| 분할수 | 40분할 | 20분할 |
| 복리 | 단리 | 수익금 즉시 반영 |
| TQQQ 매도 목표 | +10% | +15% |
| 원금 소진 대응 | 매수액 감소 | 매수액 유지 |
| 성격 | 안정적 생존 | 수익 극대화 |

> 주의: 3배 레버리지 특성상 장기 하락장에서 손실도 3배로 확대된다.
> 본 프로젝트는 전략의 수익을 보장하지 않으며, 검증·학습 목적의 시뮬레이터다.

## 토스증권 Open API 요약

- REST API. 시세 / 종목정보 / 환율 / 시장정보 / 계좌 / 보유주식 / 주문 제공
- 인증: OAuth 2.0 Client Credentials Grant → access token (Bearer)
- 계좌·자산·주문 API는 `X-Tossinvest-Account` 헤더 추가 필요
- 지원 시장: 국내(KRX), 미국 주식
- 현재 사용 범위: **캔들(일봉) 시세, 종목정보, 환율** (읽기 전용).
  주문 API는 시뮬레이터 단계에서는 사용하지 않는다.

## 범위

- **현재**: 시뮬레이터 / 백테스트만. 실제 주문 없음.
- **추후**: 실주문 연동 가능성 있음. 그때까지 주문 관련 코드는 작성하지 않는다.
