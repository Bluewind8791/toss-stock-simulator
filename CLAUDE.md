# CLAUDE.md

토스증권 Open API 기반 **라오어 무한매수법 시뮬레이터**.
프로젝트 개요·참고 링크는 [README.md](./README.md) 참조.

## 도메인 규칙 (무한매수법)

**구현 기준 버전은 v4.0.** 버전별 상세 수식은 README.md 참조.
여기엔 **구현 불변식**만 적는다.

- **T** = 누적 매수액 / 1회 매수액. 매수로 늘고, 부분 매도로 남은 수량 비율만큼 준다.
- **1회매수금** = 잔금 / (분할수 − T)  ← v4.0. 매일 변한다. 고정값 아님.
- **평단** = 누적 매수원가 / 보유수량. **매도는 평단을 바꾸지 않는다** —
  원가를 매도 수량 비율만큼 덜어낸다.
- **LOC 매수 모사** = 해당일 종가 ≤ 주문 지정가일 때만 그 종가로 체결.
- **LOC 매도 모사** = 해당일 종가 ≥ 주문 지정가일 때만 그 종가로 체결.
- **지정가 매도 모사** = 해당일 고가 ≥ 목표가일 때 목표가로 체결.
- **사이클 종료 = 보유수량 0.** 익절 여부가 아니라 수량으로 판정한다.
  3/4만 익절되고 1/4이 남으면 사이클은 이어진다.
- 하루 처리 순서는 **매도 → 매수**. 같은 날 지정가 매도 후 LOC 매수가
  체결되는 것은 규칙상 정상이다.
- 그날 주문 가격은 **전일까지의 상태**로 산출한다. 당일 체결 결과를 반영하지 않는다.

**하드코딩 금지 — 파라미터로 노출**: 버전, 분할수, 대상 종목, 시드, 단·복리 여부.

## 토스 API 사용 원칙

- 인증: OAuth 2.0 Client Credentials → Bearer access token
- 현재 단계는 **시뮬레이터 = 읽기 전용**. 주문 생성/수정/취소 API는 호출하지 않는다.
  (추후 실주문 연동은 별도 단계 — 지금은 범위 밖)
- 자격증명(client id/secret)은 `.env`에만. 커밋 금지.
- 시세 조회는 캐시/로컬 저장 우선 — 백테스트 반복 실행 시 API 재호출 최소화.

## 코드 규칙

- **Node 24 + TypeScript, 런타임 의존성 0.** 새 패키지 추가 전 내장 모듈로 되는지 먼저 확인
  (`node:sqlite`, `node:test`, `fetch` 이미 사용 중).
- 빌드 스텝 없음. Node 가 `.ts` 를 직접 실행하므로 **import 경로에 `.ts` 확장자를 붙인다.**
  `enum`/`namespace`/파라미터 프로퍼티 등 지워지지 않는 문법 금지 (`erasableSyntaxOnly`).
- 시뮬레이션 엔진(순수 계산)과 API 클라이언트·DB(I/O)는 분리한다.
  엔진은 캔들만 받아 동작 — 네트워크·DB 없이 테스트 가능해야 함.
- **백테스트와 페이퍼 트레이딩은 같은 엔진을 쓴다.** 백테스트는 캔들을 루프로,
  페이퍼는 하루 1개씩 주입. 엔진 안에 "실시간" 분기를 두지 말 것.
- 금액 계산은 부동소수점 오차에 주의. 수량은 정수.
- 검증: `npm run typecheck && npm test`

## 상시 운영 (페이퍼 트레이딩) 시 지킬 것

- **멱등성** — 같은 날짜를 두 번 처리해도 상태가 변하지 않아야 한다.
- **영속화** — 상태를 메모리에만 두지 않는다. 매 step 마다 DB 커밋. 재시작 시 갭 자동 보충.
- **시각** — 서머타임 계산을 직접 하지 말고 토스 시장 캘린더 API 로 개장일을 확인한다.

## 미확정 사항

- **v4.0 "아래로 단계별 추가 LOC매수"** — 출처에 계단 수·간격이 없다.
  현재는 전반전 2주문(별지점 아래 / 평단)만 구현. 계단이 늘면 하락일 매수량이
  달라지므로 결과에 영향이 있다.
- **v2.2 부분 매도 후 T 처리** — 공개 자료에 없음. v4.0 은 명시돼 있어서 v4.0 을 먼저 구현했다.
  v2.2 를 만들 때 다시 확인할 것.
- **v2.1 세부 수식** 미확정. 2차 출처엔 "전·후반전 분리 + 일부 물량 +5% 익절"까지만 있음.
- 원저작자 공식 채널은 네이버 카페 '무한매수법 & 밸류리밸런싱 공부모임' (로그인 필요).
  나무위키는 403 으로 못 읽는다.
- 토스 API에 **LOC 주문 타입이 문서상 없음**. 실주문 연동 시 대체 방식 필요.
